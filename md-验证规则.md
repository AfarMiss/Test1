# design-doc-analysis Markdown 验收视图规则

本文件定义 `generate_design_md.py` 生成 `all-design-md-v3/` 的结构规则。规则只基于 Excel 原生结构证据，用于给 AI 稳定解析 Markdown 验收视图。

## 1. 权威文件

- `md-list.md`：输入清单、保护清单和当前验证范围。
- `generate_design_md.py`：Markdown 生成脚本。
- `all-design-md-v3/`：当前生成结果。
- `md-golden/`：已验收保护结果。
- `check_md_golden.py`：保护校验脚本。

## 2. 基本约束

- 只读取命中的主需求 sheet；主 sheet 仍按 `界面&功能,界面,功能` 优先级匹配。
- 只使用 Excel 原生结构证据：单元格文本、合并区域、样式、行列占用、图片锚点和媒体资源。
- 不使用文件名、标题文本、界面名、功能名等业务语义作为结构判断条件。
- 不依赖固定行距、列距、上下距离或左右距离阈值做强制归属。
- 不读取未命中的 sheet 补充内容。
- 不输出调试信息、trace、sheet 名、列坐标、行坐标、锚点、source refs 或内部字段。
- 当结构证据不足时，保守输出普通列表、表格或图片项，不强行推断父子关系。

## 3. 资源与路径

- 图片必须从命中的主 sheet 对应的 xlsx drawing/media 导出并转换为 WebP。
- Markdown 图片链接使用同目录相对路径，例如 `./image_1.webp`。
- 输出目录名基于源文件 stem 清洗：替换 Windows 非法路径字符，并裁掉首尾空格与首尾下划线。
- 图片文件必须复制到对应 Markdown 输出目录；不得使用绝对路径。

## 4. Markdown 输出格式

- 使用 Markdown 标题表达 section。
- 普通阅读流使用列表项，每个 Excel 非空行原则上保留为独立 Markdown 行。
- 表格必须作为列表项输出：首行为 `- | ... |`，后续表格行使用两个空格缩进。
- 图片归属某个标题、阅读流或表格时，优先输出为列表图片项，例如 `- ![image 1](./image_1.webp)`。
- 独立图片段与相邻普通文本之间保留空行。
- 列表图片项属于当前阅读流，不在上一条列表项和图片项之间额外插入空行。
- 不使用 `|` 把多列文本拼成调试格式；只有真正表格块可以使用 Markdown table。

## 5. 标题与 section 归属

- 标题按单元格级判断，不按整行判断。
- 标题依据来自样式差异、合并区域、列带、后续内容流和图片组；这些信号可组合，不要求同时具备。
- 同一行的多列标题可分别开启独立阅读流。
- 标题只截断其所在列带或视觉组，不默认截断同一行或右侧其它列带。
- 多列并列标题按合并跨度和列带分组；合并跨度有重叠时属于同一纵向阅读流。
- 不能仅凭“单个强调单元格、下方有内容”提升为全局 section；必须结合后续视觉组、卡片组或明确列带重启。
- 当常规样式规则没有识别出标题时，可用保守兜底：短文本、左侧单元格、样式稀有或突出、下方存在连续内容。
- 下一标题位于左侧图片列带，而右侧说明流列带保持一致且在下一标题首图之前已连续出现时，右侧说明流可继续归属上一标题。
- 下一标题下的编号文本若与该标题首图同行或位于首图之后，则归属下一标题，不回流上一标题。
- 旧右侧标题不能跨越后续左侧视觉主体图片持续吞并规则文本；落在新视觉主体图锚点行段内的右侧说明应归属更近的视觉 section。

## 6. 阅读流

- 同一 Excel 行内，结构上属于同一条文本的短前缀单元格和正文单元格应合并为一行。
- 前缀判断不得依赖具体业务词；列表符号、数字序号和短前缀只作为结构信号。
- 多列并列内容必须拆成阅读流或表格；不能简单按 Excel 行顺序交错输出。
- 普通阅读流不输出列坐标或内部定位信息。

## 7. 表格与网格

- 多列重复结构、左右规则列、并列控件说明、对照说明优先输出为 Markdown table。
- 表格默认保留 Excel 行级左右对应关系；不能仅因某列中间有空单元格就把左右列各自压缩后重新配对。
- Excel 物理空行只作为分组证据，不输出为空 table row。
- 只有结构已判定左右列是独立阅读流时，才允许左右列各自忽略空行后连续输出。
- 表格列带按当前局部内容计算，不使用整张 sheet 的全局列带。
- 表格中右侧续写列若未出现在表头行，且位于已有表头列右侧，应并入左侧最近表头列。
- 某列先作为短前缀列与右侧正文列合并时，后续正文列单独出现仍归属同一逻辑列。
- 连续内容块首行是多个并列表头时，即使后续正文只落在某个表头列带，也保留 table；未使用表头列为空。
- 表格首行只有右侧内容、第二行同时有左右内容时，说明右侧流提前开始；可将第二行左侧内容上移，避免左右标题错位。
- 左侧普通列表与右侧多列表格并列时，应拆成“左侧列表 + 右侧表格”两个同级块，不合成大表格。
- 三列及以上表格中，只出现一次、位于左右主列之间、不形成连续正文流的短辅助列，可并入左侧最近主列。
- 上述短辅助列并入只适用于中间列；最右侧只出现一次的短辅助列保留为辅助列证据，不并入右侧主列。
- 三列及以上表格中，某列只有一个短图片标题或标签且无其它内容支撑时，不生成独立 Markdown 表格列。
- 以冒号结尾的短标签是字段行，不作为图片标题删除；以列表标记开头的行是正文，不作为图片标题删除。
- 若单张左图对应的正文区只有首行同时出现左右两列、后续多数行仅落在右列，但 `table_matrix` 能稳定压成两列，则视为弱 table 结构，可输出为 `图片 + 两列表格`。
- 单图两列表格中若嵌入连续多列（≥3 列）同行行，且连续行列结构一致，应拆为"两列表格 + 嵌入多列表格"两个同级块；嵌入多列表格用 `raw_rows` 字段行检测提取。
- 嵌入多列表格一旦由连续多列行建立列带，后续只落在这些已建立列带中的稀疏行，仍归入该嵌入表格；同一行中不属于该列带的左侧辅助列继续留在外层说明表。

## 8. 图片归属与输出顺序

- 图片位置必须结合锚点行列、相邻文本、标题列带和局部图文组决定，不能统一追加到 section 开头或末尾。
- 图片作为视觉主体时优先输出图片，再输出对应说明或表格。
- 左侧主视觉图不要求严格位于首个文字列带左边；图片右边界与首个文字列带相接，且后方仍有向右展开的说明流时，仍视为左侧主视觉图。
- 左侧视觉主体图片对应右侧整体表格时，若表格开头若干行位于图片锚点上方，但列带与图片右侧后续表格一致，仍保持为同一个表格块；输出顺序为图片在前、完整表格在后。
- 当图片上方最近的非空多列表头行与后续表格列带对齐，且中间无其它非空文本时，不提前输出为普通列表。
- 单张左图旁若存在一个局部只出现一次的短辅助列，且更右侧存在连续说明流，可输出为 `图片 + 两列表格`；短辅助列为左列标签，连续说明流为右列正文。
- 上述单图 table 只在图片前没有已成形表格前半段时启用；如果图片前同一 section 已有多列成对表格行，不把图片后的后半段改写成新的单图 table。
- 单图 table 的短辅助列可与主图右边界存在很小列间隔，但必须仍属于同一局部图文组。
- 单图 table 中，图片上一行紧贴出现单列说明，且与图片后的连续说明流连成同一组时，该行可并入 table 正文首段。
- 右侧辅助图若锚点起始行对应某条文本行，应贴到该行后；无法对应单行时，作为连续文本组后的辅助图输出。
- 右侧辅助图若实际落在后续左侧视觉主体图的垂直范围内，且该主图位于辅助图左侧，则归属后续视觉 section。
- 同一标题区域内，多张图片处于同一左侧视觉列带，且右侧说明流不以新编号重新开始时，组成连续视觉主体组：先按视觉顺序输出图片，再输出右侧说明。
- 后一张图片对应的右侧说明流在同一位置以新编号重新开始时，视为新的图文组。
- 同一标题区域内，多张左侧视觉主体图片纵向排列，且每张图片的锚点行段右侧都形成独立 table 时，应按视觉顺序输出为多个 `图片 + 局部 table` 图文组，不合并为父级整体 table。
- 后一张图片有独立标题，但其后仍存在上一张图片所属父级整体说明或表格时，该标题和图片作为父级下嵌套列表块，不使用子标题截断父级正文。

## 9. 图片标题、alt 与隐藏文本

- 图片锚点同行或上一行左侧存在独立短文本，且该文本不是编号、列表前缀或正文前缀时，可作为图片小标题。
- 图片上方或下方相邻行存在与图片数量匹配、列带对齐的短文本时，该短文本作为 Markdown 图片 alt；单张和多张图片都适用。
- 已作为 alt 的短文本不再作为普通正文重复输出。
- 同一表格单元格或列表项内出现多张图片时，按视觉顺序输出：先从左到右，再从上到下；不能按图片编号或锚点起始行排序。
- 被图片锚点矩形覆盖的底层单元格默认不作为正文输出。
- 被覆盖单元格在以下情况可保留：图片起始行短标题候选、编号或列表前缀、右侧存在同文重复的结构性文本。
- 被覆盖单元格同一行右侧仍存在图片外说明流，且不满足可保留条件时，视为底层残留，不进入父级 table/list。

## 10. 子级图片与嵌套块

- 当父级 section 中存在只承载图片的子级标题，且后续仍有父级整体表格或说明时，子级标题使用父级列表项加嵌套图片表达；后续父级内容继续作为同级块输出。
- 带独立图片标题的子级图片只吸收与图片右边界相邻的局部文字列带。
- 如果同一行更右侧列带属于父级规则或表格，应按单元格项拆分，不能整行归入子级图片。
- 当相邻文字列带与更右侧列带在多行持续成对出现，说明它们是同一个父级表格左右列，不能把其中一列拆成子级图片说明。
- 如果候选子级图片说明所在列带在图片前后持续作为父级表格的非最左列出现，则该文字归属父级表格，子级图片标题下只输出图片。

## 11. 卡片组

- 同一局部区域存在重叠图片组，并且图片组附近有重复短标签行时，优先输出为阅读型卡片组。
- 卡片组按视觉顺序从左到右、从上到下输出。
- 每个卡片按“小标题、列表图片项、字段列表”输出。
- 同一图片上方且归属该图片的文本作为卡片小标题。
- 横向图片组上方多个文本单元格若分别对齐不同图片列带，则按多个“小标题 + 列表图片项”卡片输出，不合并为横向图片行。
- 横向卡片组上方若先出现一个位于当前 section 左边界的单个强调单元格，随后才出现多个对齐图片列带的卡片标题行，则该单元格是新组标题。
- 能稳定识别卡片组时，不输出空洞大表格。

## 12. 验收流程

- 通过样本以 `md-golden/manifest.txt` 为准。
- 样本通过后，从 `md-list.md` 的未保护清单移入已保护清单。
- 将对应输出目录复制到 `md-golden/`，并更新 `manifest.txt`。
- 重新运行 `python .codex/design-doc-analysis/generate_design_md.py`。
- 再运行 `python .codex/design-doc-analysis/check_md_golden.py`，确保 protected 样本、manifest 和生成时间一致。
