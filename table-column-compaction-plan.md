# Table 列内上移压缩预备计划

## 当前状态

未启用。

## 目标

对阅读型 Markdown 表格做可选压缩：当左右列是独立阅读流、存在大量单侧空单元格时，允许把同列后续内容上移到空位中，减少稀疏行。

示例：

```md
| 显示职业和职业图标，需要有选中效果 | 2 角色立绘或模型 |
|  | 显示默认选择的职业和性别的模型。 |
|  | 默认选择第一个职业 |
| 4 姓名输入框 |  |
| 角色姓名和输入框 | 3 角色职业信息框 |
```

可读压缩为：

```md
| 显示职业和职业图标，需要有选中效果 | 2 角色立绘或模型 |
| 4 姓名输入框 | 显示默认选择的职业和性别的模型。 |
| 角色姓名和输入框 | 默认选择第一个职业 |
```

## 风险

大量策划案存在左右行语义对应的表格，例如 2、5、15、16、18、19、21、28 等。全局启用会制造错误的一一对应关系。

## 启用前条件

- 只对明显独立阅读流启用。
- 不跨编号段边界。
- 不对高比例左右同行都有内容的表格启用。
- 必须先 dry-run 全量输出并逐项查看保护项差异。

## 当前优先级

低。先修复确定性问题：标题识别、section 归属、表格列基准。
